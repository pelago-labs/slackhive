---
title: "Install a Slack App"
description: "Create and configure a Slack app for your SlackHive agent, including all required scopes and Socket Mode."
---

Each SlackHive agent needs its own Slack app. This gives each agent a unique identity, avatar, and @mention handle in your workspace.

## Overview

You'll create a Slack app from a manifest (a JSON config that pre-fills all scopes and settings), install it to your workspace, and copy three credentials into SlackHive.

## Step 1: Generate the manifest

The easiest path is to generate the manifest from inside the agent creation wizard (**New Agent → Step 2: Slack App → Generate Manifest**). SlackHive builds a manifest with your agent's name and all required OAuth scopes pre-configured.

If you're creating the app outside of the wizard, use this base manifest and substitute your agent's name:

```json
{
  "display_information": {
    "name": "Your Agent Name"
  },
  "features": {
    "bot_user": {
      "display_name": "Your Agent Name",
      "always_online": true
    }
  },
  "oauth_config": {
    "scopes": {
      "bot": [
        "app_mentions:read",
        "channels:history",
        "channels:read",
        "chat:write",
        "chat:write.public",
        "files:read",
        "files:write",
        "groups:history",
        "groups:read",
        "im:history",
        "im:read",
        "im:write",
        "mpim:read",
        "reactions:read",
        "reactions:write",
        "users:read"
      ]
    }
  },
  "settings": {
    "event_subscriptions": {
      "bot_events": [
        "app_mention",
        "message.channels",
        "message.groups",
        "message.im",
        "message.mpim"
      ]
    },
    "interactivity": {
      "is_enabled": false
    },
    "org_deploy_enabled": false,
    "socket_mode_enabled": true,
    "token_rotation_enabled": false
  }
}
```

## Step 2: Create the app

<Steps>
  <Step title="Go to api.slack.com/apps">
    Navigate to [https://api.slack.com/apps](https://api.slack.com/apps) and sign in with your Slack account.
  </Step>
  <Step title="Create from manifest">
    1. Click **Create New App**
    2. Select **From an app manifest**
    3. Choose your workspace from the dropdown
    4. Click **Next**
  </Step>
  <Step title="Paste the manifest">
    Select the **JSON** tab, clear the default content, and paste your manifest. Click **Next**.
  </Step>
  <Step title="Review and create">
    Slack shows a summary of the OAuth scopes and features. Verify the scopes match the list above, then click **Create**.
  </Step>
</Steps>

## Step 3: Install to workspace

<Steps>
  <Step title="Navigate to OAuth & Permissions">
    In the left sidebar of your app's settings page, click **OAuth & Permissions**.
  </Step>
  <Step title="Install the app">
    Click **Install to Workspace**, review the permissions, and click **Allow**.
  </Step>
  <Step title="Copy the Bot Token">
    After installation, you'll see **Bot User OAuth Token** starting with `xoxb-`. Copy this — you'll need it for SlackHive.
  </Step>
</Steps>

## Step 4: Enable Socket Mode

SlackHive uses Socket Mode (a persistent WebSocket connection) instead of webhooks — no public URL required.

<Steps>
  <Step title="Go to Socket Mode settings">
    In the left sidebar, click **Socket Mode**.
  </Step>
  <Step title="Enable Socket Mode">
    Toggle **Enable Socket Mode** to on.
  </Step>
</Steps>

## Step 5: Create an App-Level Token

App-level tokens are separate from bot tokens and authorize the Socket Mode connection.

<Steps>
  <Step title="Go to Basic Information">
    In the left sidebar, click **Basic Information**. Scroll down to **App-Level Tokens**.
  </Step>
  <Step title="Generate a token">
    Click **Generate Token and Scopes**.
    - **Token Name**: anything descriptive, e.g. `socket-mode`
    - **Scopes**: add `connections:write`
    
    Click **Generate**.
  </Step>
  <Step title="Copy the App Token">
    Copy the token starting with `xapp-`. This is your App Token.
  </Step>
</Steps>

## Step 6: Copy the Signing Secret

<Steps>
  <Step title="Go to Basic Information">
    Still on the **Basic Information** page, scroll to **App Credentials**.
  </Step>
  <Step title="Copy the Signing Secret">
    Click **Show** next to **Signing Secret** and copy the value.
  </Step>
</Steps>

## Summary: where to find each credential

| Credential | Location in Slack |
|-----------|------------------|
| **Bot Token** (`xoxb-...`) | OAuth & Permissions → Bot User OAuth Token |
| **App Token** (`xapp-...`) | Basic Information → App-Level Tokens |
| **Signing Secret** | Basic Information → App Credentials → Signing Secret |

## Troubleshooting

**Agent shows "Disconnected" status**
- Verify Socket Mode is enabled on the Slack app
- Confirm the App Token has the `connections:write` scope
- Check that the App Token starts with `xapp-` (not `xoxb-`)

**Agent doesn't respond to @mentions**
- Confirm the app is installed to the workspace (OAuth & Permissions → Install to Workspace)
- Make sure the bot is invited to the channel (`/invite @agent-name`)
- Verify `app_mentions:read` and `channels:history` scopes are present

**"invalid_auth" error in logs**
- The Bot Token may have been regenerated after a reinstall — update it in SlackHive

**Bot doesn't see DMs**
- Ensure `im:history`, `im:read`, and `im:write` scopes are included
- You may need to reinstall the app after adding scopes
